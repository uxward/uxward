Brownfield. The site shipped before any formal requirements existed; this master PRD is the technical glue over the existing build (see `000-tech-audit.md`) and the strategy/design layer (`A-Product-Brief/`, `B-Trigger-Map/`, `C-UX-Scenarios/`, `D-UX-Design/`). It connects those — it does not repeat them.

## Architecture Overview
Static Astro v6 site, file-based routing, single `Base.astro` layout, one authoritative token stylesheet (`src/styles/global.css`). The only data-driven surface today is the case slate (`src/data/slate.js`). Per-page styling is scoped `<style>` blocks. Full architecture: `000-tech-audit.md`.

## Cross-Feature Dependencies
- **New data source `src/data/essays.js`** (mirrors the `slate.js` pattern) becomes the single source of truth for essay metadata. Three surfaces depend on it once it exists: the Writing index (list + count), the Work index pivot count, and the Home "recent writing" cards. Build `essays.js` **first** — the three consumers depend on it.
- **`src/pages/index.astro` is edited by multiple requirements** (fold deck, fold height, Speakeazy domain, recent-writing cards). These must be implemented and committed in sequence on the same file to avoid conflicts.
- The Speakeazy live URL (`speakeazy.pro`) is referenced by both the home page and the case page — standardize in one pass.

## Technical Decisions
- **Verification is browser-only** — no test suite exists in the repo (confirmed 2026-06-09). Each requirement is verified by opening the live page.
- **Essay drift fixed at the source (option a)** — `essays.js` replaces hardcoded essay lists/counts rather than correcting literals in place.
- **Deferred (owner's call, not in scope):** Google-Fonts-CDN → self-host, and the "Cloudflare" footer claim with no adapter. Tracked in the tech audit only.
- **Asset-blocked, not built:** the About portrait and the Speakeazy case artifact images — no source images exist; tracked as blocked requirements.

## Open Technical Questions
- None blocking. Deployment target (footer says Cloudflare; no adapter in repo) is unverified but does not affect the in-scope requirements.
